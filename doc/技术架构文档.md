# Stock Foker - 技术架构文档

## 1 技术栈

| 层级 | 技术方案 | 版本 | 说明 |
| ------ | --------- | ------ | ------ |
| 前端框架 | React + Vite + TypeScript | React 19, Vite 6 | 现代化 SPA |
| UI 组件库 | Ant Design | 5.x | 数据密集型组件（表格/表单/布局） |
| 图表 | ECharts (echarts-for-react) | 5.x | K 线图、成交量、技术指标、雷达图可视化 |
| HTTP 客户端 | Axios | 1.x | 前端 API 调用 |
| 路由 | React Router | 7.x | SPA 路由管理 |
| 后端框架 | Python FastAPI | 0.135+ | 高性能异步 API |
| ORM | SQLAlchemy | 2.0 | 数据库操作 |
| 数据库 | SQLite | - | 本地持久化，零运维 |
| 行情数据 | AKShare（主）+ 新浪财经 API（备） | - | 双数据源容灾 |
| 技术指标 | pandas-ta | 0.4.x | MACD/KDJ/RSI/布林带等计算 |
| 数据处理 | pandas | 3.x | 数据清洗与转换 |
| LLM 客户端 | httpx | 0.28+ | OpenAI 兼容 Chat Completions API 调用 |
| 环境配置 | python-dotenv | 1.x | .env 文件加载 LLM 配置 |

## 3 数据库设计

### 3.1 focus_stock - 股票关注表

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | INTEGER PK | 自增主键 |
| stock_code | VARCHAR(10) | 股票代码 |
| stock_name | VARCHAR(50) | 股票名称 |
| time_frame | ENUM | short/medium/long |
| is_active | INTEGER | 1=当前关注, 0=历史 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 trade_records - 交易记录表

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | INTEGER PK | 自增主键 |
| stock_code | VARCHAR(10) | 股票代码 |
| stock_name | VARCHAR(50) | 股票名称 |
| trade_type | ENUM | buy/sell |
| price | FLOAT | 成交价格 |
| quantity | INTEGER | 成交数量 |
| reason | TEXT | 买卖理由 |
| market_sentiment | ENUM | optimistic/neutral/pessimistic |
| target_price | FLOAT | 目标价位 |
| expected_hold_days | INTEGER | 预期持有天数 |
| actual_result | FLOAT | 实际盈亏 |
| result_note | TEXT | 结果备注 |
| traded_at | DATETIME | 成交时间 |
| record_mode | VARCHAR(8) | 记录模式 simple/full |
| created_at | DATETIME | 创建时间 |

### 3.3 kline_cache - K线数据缓存表

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | INTEGER PK | 自增主键 |
| stock_code | VARCHAR(10) | 股票代码（索引） |
| period | VARCHAR(10) | K线周期 daily/weekly/monthly |
| date | VARCHAR(10) | 日期 YYYY-MM-DD |
| open | FLOAT | 开盘价 |
| close | FLOAT | 收盘价 |
| high | FLOAT | 最高价 |
| low | FLOAT | 最低价 |
| volume | FLOAT | 成交量 |
| turnover | FLOAT | 成交额 |

唯一约束: (stock_code, period, date)

### 3.4 stock_positions - 持仓表

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | INTEGER PK | 自增主键 |
| stock_code | VARCHAR(10) | 股票代码（唯一） |
| stock_name | VARCHAR(50) | 股票名称 |
| cost_price | FLOAT | 成本价 |
| quantity | INTEGER | 持仓数量 |
| take_profit_price | FLOAT | 止盈价 |
| stop_loss_price | FLOAT | 止损价 |
| first_buy_date | DATETIME | 首次买入日期 |
| note | TEXT | 备注 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.5 agent_result_cache - Agent 结果缓存表

| 字段 | 类型 | 说明 |
| ------ | ------ | ------ |
| id | INTEGER PK | 自增主键 |
| agent_name | VARCHAR(50) | Agent 名称（索引） |
| stock_code | VARCHAR(10) | 股票代码（索引） |
| cache_key | VARCHAR(100) | 缓存键（日期） |
| status | VARCHAR(20) | 状态 success/degraded/error |
| llm_used | INTEGER | 是否使用了 LLM（0/1） |
| data | TEXT | JSON 序列化的结果数据 |
| error_message | TEXT | 错误信息 |
| created_at | DATETIME | 创建时间 |

唯一约束: (agent_name, stock_code, cache_key)

### 4.4 炒股画像

| 方法 | 路径 | 说明 |
| ------ | ------ | ------ |

### 4.5 持仓管理

| 方法 | 路径 | 说明 |
| ------ | ------ | ------ |
| GET | /api/positions/{stock_code} | 获取指定股票持仓 |

### 4.6 Agent 链路（AI 分析）

| 方法 | 路径 | 说明 |
| ------ | ------ | ------ |
| POST | /api/agent/sentiment | 消息面情绪分析（body: stock_code, stock_name） |
| POST | /api/agent/sector | 板块联动分析 |
| POST | /api/agent/macro | 宏观环境分析 |

| GET | /api/agent/llm-status | LLM 配置状态 |
| POST | /api/agent/reload-config | 热重载 .env 配置（无需重启） |
| DELETE | /api/agent/cache/{stock_code} | 清除指定股票的 Agent 缓存 |

## 5 数据流架构

### 5.1 行情数据流

```text
用户操作 → 前端页面 → Axios API → FastAPI 路由
                                      ↓
                              stock_service.py
                              ┌──────────────────┐
                              │ 1. 查询本地缓存    │
                              │    ↓ (缓存命中)    │
                              │    直接返回        │
                              │    ↓ (缓存缺失)    │
                              │ 2. AKShare（主）    │
                              │    ↓ (失败)        │
                              │ 3. 新浪财经 降级    │
                              │    ↓               │
                              │ 4. 写入 SQLite 缓存 │
                              └──────────────────┘
                                      ↓
                              calculate_indicators()
                              (pandas-ta 计算技术指标)
                                      ↓
                              generate_advice()
                              (多指标综合评分 + 推理)
                                      ↓
                              返回前端渲染
```

### 5.2 Agent 链路数据流

```text
用户请求 AI 分析 → agent_router.py
                        │
       ┌────────────────┼────────────────┐
       │                │                │
       ▼                ▼                ▼
 Sentiment Agent   Sector Agent    Macro Agent
  (消息面情绪)      (板块联动)      (宏观环境)
       │                │                │
       │  data_fetcher.py 抓取 AKShare 数据
       │  base_agent.execute() 模板方法：
       │  fetch_data → build_prompt → call_llm → parse_response
       │  (LLM 不可用时自动走 fallback 降级逻辑)
       │                │                │
       └────────────────┼────────────────┘
                        │ ThreadPoolExecutor 并行
                        ▼
              Enhanced Advice Agent
              + 技术指标 (advice_service)
              + 用户画像 (profile_service)
              + 持仓信息 (stock_positions)
                        │
                        ▼
              agent_result_cache 缓存
                        │
                        ▼
                 综合建议响应
```

## 6 启动与部署

```bash
# 启动（自动检查依赖、build）
./start.sh

# 终止
./stop.sh

# 手动启动
cd backend && source venv/bin/activate && uvicorn app.main:app --port 8000
cd frontend && npm run dev
```

- 后端: <http://127.0.0.1:8000>
- 前端: <http://127.0.0.1:5173>
- 前端 Vite 配置了 /api 代理到后端
- 日志输出: `logs/backend.log`, `logs/frontend.log`

## 7 LLM 配置

通过 `backend/.env` 文件配置 LLM（参考 `.env.example`）：

| 配置项 | 说明 | 默认值 |
| ------ | ------ | ------ |
| LLM_ENABLED | AI 功能总开关 | false |
| LLM_API_KEY | API 密钥 | (用户填写) |
| LLM_BASE_URL | API 基地址 | `https://api.openai.com/v1` |
| LLM_MODEL | 模型名称 | gpt-3.5-turbo |
| LLM_TEMPERATURE | 温度参数 | 0.3 |
| LLM_MAX_TOKENS | 最大输出 token | 2048 |
| LLM_TIMEOUT | 请求超时(秒) | 60 |
| LLM_ENABLE_THINKING | 模型思考模式开关 | false |

支持所有 OpenAI 兼容 API 的模型：DeepSeek、通义千问、Moonshot、智谱 GLM 等。修改配置后可通过 AI 设置页"重新加载配置"按钮热生效，无需重启后端。
