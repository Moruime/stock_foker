## 2 批量导入审查

| 检查项 | 结果 | 说明 |

| 文件编码处理 | 通过 | GBK 优先，UTF-8 降级 |

## 3 批量删除审查

### 3.1 端点 `POST /api/trades/batch-delete`

| 检查项 | 结果 | 说明 |
| --- | --- | --- |
| 空 ids 校验 | 通过 | 返回 400 |
| backfill 记录删除 | 通过 | 直接删除 |
| realtime 买入反向调整 | 通过 | 加权成本重算 |
| realtime 卖出反向调整 | 通过 | 加回持仓数量 |
| 持仓清零处理 | 通过 | 删除持仓记录 |
| 事务原子性 | 通过 | 单次 commit |
| 返回值完整性 | 通过 | deleted + adjusted + total |

### 3.2 与单条删除一致性对比

| 逻辑点 | 单条删除 | 批量删除 | 一致 |
| --- | --- | --- | --- |
| backfill 直接删除 | 是 | 是 | 是 |
| realtime 买入减仓 | 是 | 是 | 是 |
| realtime 卖出加仓 | 是 | 是 | 是 |
| 持仓清零删持仓 | 是 | 是 | 是 |
| 成本价重算公式 | 加权平均 | 加权平均 | 是 |

### 3.3 潜在风险

- **同股票多条删除顺序**：批量删除同一股票的
  多条 realtime 记录时，循环内每次读取最新
  position 状态，逻辑正确但依赖 SQLAlchemy
  Session 内的脏读一致性（当前 SQLite 单线程
  无并发问题）
