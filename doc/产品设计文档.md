# Stock Foker - 个人股票分析应用产品设计文档

## 1 产品概述

### 1.1 产品定位

Stock Foker 是一款面向个人投资者的自用股票分析应用，核心理念是 **"深度聚焦、数据驱动、自我进化"**。通过单股票聚焦模式，融合技术面、基本面、消息面数据，结合用户个人交易画像，提供个性化的辅助决策建议。

### 1.2 核心价值

- **减少信息噪音**：一次聚焦一支股票，避免精力分散
- **多维度分析**：技术指标 + 板块联动 + 消息面的全方位数据融合
- **自我认知**：通过操作记录与炒股画像，帮助用户认识自己的交易风格和弱点
- **闭环决策**：数据输入 → 用户画像 → 智能决策辅助 → 复盘优化

### 1.3 目标用户

个人投资者（自用），具备一定股票交易经验，希望通过数据和工具提升交易决策质量。

## 2 功能模块设计

### 2.1 股票关注管理

#### 2.1.1 单股票聚焦模式

- 用户可设置当前关注的股票（一次仅关注一支）
- 所有分析、建议围绕当前关注的股票展开
- 支持快速切换关注股票

#### 2.1.2 时间框架设置

- 用户可设置当前关注的交易模式：**短线 / 中线 / 长线**
- 不同时间框架下，技术指标参数、分析逻辑、建议策略自动调整

#### 2.1.3 历史关注记录

- 切换股票后，历史分析记录和操作数据不丢失
- 提供历史关注列表和归档机制，支持回溯查看

### 2.2 技术面分析

#### 2.2.1 K 线与走势展示

- 日 K / 周 K / 月 K 线图
- 分时走势图
- 支持缩放和区间选择

#### 2.2.2 核心技术指标

| 指标类别 | 包含指标 |
| --------- | --------- |
| 趋势类 | MA（5/10/20/60）、MACD、布林带 |
| 震荡类 | KDJ、RSI |
| 量价类 | 成交量、换手率、量比 |
| 资金类 | 主力资金流向、北向资金 |

#### 2.2.3 对比基准

- 个股 vs 大盘（沪深300/上证指数）涨跌对比
- 个股 vs 所在板块内排名与相对表现

### 2.3 板块联动分析

- 显示当前股票所属板块（行业板块 + 概念板块）
- 板块整体走势与关键指标
- 板块内资金流入/流出情况
- 板块轮动趋势分析

### 2.4 消息面数据

#### 2.4.1 数据来源

| 数据类型 | 来源 | 更新频率 |
| --------- | ------ | --------- |
| 公司公告 | 巨潮资讯 | 实时 |
| 财务报表 | 交易所/Tushare | 季度 |
| 财经新闻 | 东方财富/同花顺 | 每日 |
| 研报摘要 | 券商研报 | 不定期 |

#### 2.4.2 消息过滤与情绪分析

- 通过 NLP 对新闻进行 **多空情绪评分**，过滤低质量噪音
- 区分"有效消息"和"噪音"，突出显示高影响力信息
- 消息时间线展示，关联股价走势

### 2.5 宏观环境感知

- 大盘整体趋势（牛/熊/震荡判断）
- 关键宏观指标：利率政策、M2、社融数据
- 市场整体情绪指标（涨跌家数比、涨停跌停数等）
- 宏观环境对当前操作建议的影响权重

### 2.6 操作记录与炒股画像

#### 2.6.1 结构化操作记录

每笔操作需记录以下字段：

| 字段 | 说明 |
| ----- | ------ |
| 操作类型 | 买入 / 卖出 |
| 价格与数量 | 成交价、成交量 |
| 买入/卖出理由 | 结构化选择 + 自由文本 |
| 当时市场情绪判断 | 乐观 / 中性 / 悲观 |
| 预期目标 | 目标价位、预期持有周期 |
| 最终结果 | 盈亏金额/比例、与预期偏差 |

#### 2.6.2 炒股画像维度

基于操作记录，自动生成以下画像维度：

- **持仓周期偏好**：实际持仓天数分布（短线/波段/中长线）
- **风险偏好**：止损幅度、最大回撤容忍度
- **交易频率**：日均/周均交易次数
- **盈亏比分布**：平均盈利 vs 平均亏损比
- **擅长行情类型**：趋势行情 vs 震荡行情的胜率对比
- **情绪化交易倾向**：追涨杀跌频率、连续亏损后的操作偏差
- **认知偏差分析**：高估/低估自身判断准确率的程度

### 2.7 风险控制模块

#### 2.7.1 仓位管理建议

- 基于账户总资金和风险偏好，建议单笔投入比例
- 提供分批建仓/减仓的方案建议
- 当前持仓的整体风险暴露评估

#### 2.7.2 止盈止损机制

- 根据技术指标和画像，建议止损位和止盈位
- 支持用户自定义止损止盈规则
- 价格触达时主动提醒

#### 2.7.3 风险预警

- 单日 / 单周最大亏损提醒
- 连续亏损后的 **冷静期提示**（建议暂停交易）
- 异常波动预警（个股或板块异动）

### 2.8 量化交易建议

#### 2.8.1 辅助决策模式

定位为 **辅助决策**，不做自动下单执行，降低风险。

#### 2.8.2 建议内容

- 买入 / 卖出 / 持有信号
- 建议仓位比例
- 建议价位区间

#### 2.8.3 可解释性（核心要求）

每条建议必须附带推理过程，例如：

> **建议：减仓 30%**
> 理由：RSI 超买（82）+ 消息面偏空（近3日负面新闻占比 70%）+ 你的历史数据显示此类情况下持有亏损概率为 65%

#### 2.8.4 策略模型

- 基于技术指标的信号策略（均线、MACD 金叉死叉等）
- 结合用户画像的个性化权重调整
- 综合消息面情绪的多因子评分

### 2.9 智能选股推荐

- 基于用户炒股画像（擅长的行情类型、风险偏好等）筛选标的
- 结合当前市场环境过滤（熊市不推荐高波动票）
- 覆盖个股和 ETF
- 推荐理由说明（为什么这支适合你）

### 2.10 复盘模块

#### 2.10.1 单笔交易复盘

- 还原当时的市场数据和操作上下文
- 事后分析：买对了还是卖错了，原因是什么
- 与"如果不操作"的对比分析

#### 2.10.2 周期性复盘报告

- 周报 / 月报自动生成
- 盈亏统计、胜率变化趋势
- 与大盘对比的超额收益分析
- 画像变化趋势（交易习惯是否在改善）

### 2.11 回测功能

- 对推荐策略进行历史数据回测
- 展示策略在不同行情周期下的表现
- 支持自定义策略参数调整后重新回测

### 2.12 通知与提醒

- 价格触达目标位提醒
- 重要公告发布提醒
- 技术指标信号触发提醒
- 止损止盈位触达提醒
- 推送方式：应用内通知（后续可扩展微信/邮件）

## 3 数据架构

### 3.1 数据源规划

| 数据类型 | 推荐数据源 | 备选方案 |
| --------- | ----------- | --------- |
| 行情数据（K线/实时） | AKShare | Tushare、东方财富接口 |
| 财务数据 | Tushare | AKShare |
| 新闻/公告 | 东方财富、巨潮资讯 | 同花顺 |
| 板块数据 | AKShare | 东方财富 |
| 宏观数据 | AKShare | 国家统计局接口 |

### 3.2 数据更新策略

- **实时数据**：交易时段内分钟级更新（分时走势、资金流向）
- **日级数据**：每日收盘后更新（日K线、技术指标、板块数据）
- **低频数据**：按事件触发更新（财报、公告、研报）
- **本地缓存**：历史数据本地持久化，减少重复请求

### 3.3 数据存储

- 用户操作数据、画像数据：本地数据库（SQLite）
- 历史行情缓存：本地数据库
- 新闻/公告文本：本地文件存储 + 索引

## 4 技术选型

| 层级 | 技术方案 | 说明 |
| ----- | --------- | ------ |
| 前端 | React + Vite + ECharts | 现代化 Web 应用，ECharts 做 K 线和图表可视化 |
| 后端 | Python + FastAPI | 方便集成数据分析库（pandas/numpy）和量化库 |
| 数据库 | SQLite | 个人使用场景足够，无需额外运维 |
| AI/NLP | 大模型 API | 新闻情绪分析、报告生成、智能建议推理 |
| 部署 | 本地部署 | 保护个人交易数据隐私 |

## 5 版本规划

### 5.1 MVP（第一阶段）

核心目标：跑通基础数据展示 + 操作记录闭环

- 单股票关注设置（含时间框架选择）
- 基础 K 线与核心技术指标展示
- 操作记录（结构化录入）
- 基础炒股画像生成
- 简单买卖建议（基于技术指标，附推理说明）

### 5.2 第二阶段

核心目标：补全数据维度 + 风险控制

- 板块联动分析
- 消息面数据整合 + 情绪分析
- 宏观环境感知
- 风险控制模块（仓位管理、止盈止损、风险预警）
- 对比基准分析
- 通知提醒机制

### 5.3 第三阶段

核心目标：智能化升级 + 自我进化

- 量化策略优化（多因子模型）
- 智能选股推荐（个股 + ETF）
- 回测功能
- 复盘模块（单笔复盘 + 周期报告）
- 历史关注记录与归档

## 6 非功能性要求

### 6.1 隐私与安全

- 所有数据本地存储，不上传云端
- 操作记录和画像数据加密存储
- 数据源 API Key 安全管理

### 6.2 性能

- 页面首屏加载 < 2s
- K 线图交互流畅（拖拽、缩放无卡顿）
- 数据更新后 < 3s 刷新展示

### 6.3 可扩展性

- 架构设计支持未来多股票并行跟踪
- 策略模型支持插件化扩展
- 数据源支持可替换适配

## 7 Agent 链路设计（第二阶段核心）

### 7.1 架构总览

第二阶段引入 LLM Agent 链路，将技术面、消息面、板块联动、宏观环境四个维度的数据通过 AI 融合分析，生成可解释性更强的综合建议。

```text
前端 (消息面/板块/宏观 3个新页面 + 增强版行情分析 + 设置页)
                    │ HTTP
             agent_router.py
                    │
          ┌─────────┴──────────┐
          │     Agent Layer     │
          │                     │
          │  ┌───────────────┐  │
          │  │ Sentiment Agent│──┐
          │  │ (消息面情绪)    │  │
          │  └───────────────┘  │
          │  ┌───────────────┐  │  并行执行
          │  │ Sector Agent  │──┤
          │  │ (板块联动)     │  │
          │  └───────────────┘  │
          │  ┌───────────────┐  │
          │  │ Macro Agent   │──┘
          │  │ (宏观环境)     │
          │  └───────────────┘
          │          │
          │          ▼ 汇总
          │  ┌───────────────┐
          │  │Enhanced Advice │← 技术指标 + 用户画像
          │  │ Agent (综合建议)│
          │  └───────┬───────┘
          └──────────┼─────────┘
                     │
            llm/client.py
          (OpenAI 兼容 API)
                     │
               .env 配置文件
     (LLM_API_KEY / LLM_BASE_URL / LLM_MODEL)
```

### 7.2 LLM 抽象层

#### 7.2.1 设计原则

统一走 OpenAI-compatible Chat Completions API。国内的 DeepSeek、通义千问、Moonshot、智谱 GLM 等全部提供兼容接口，只需切换 `base_url` 和 `api_key`。

#### 7.2.2 配置项

| 配置项 | 说明 | 默认值 |
| ----- | ------ | ----- |
| LLM_ENABLED | AI 功能总开关 | false |
| LLM_API_KEY | API 密钥 | (用户填写) |
| LLM_BASE_URL | API 基地址 | <https://api.openai.com/v1> |
| LLM_MODEL | 模型名称 | gpt-3.5-turbo |
| LLM_TEMPERATURE | 温度参数 | 0.3 |
| LLM_MAX_TOKENS | 最大输出 token | 2048 |
| LLM_TIMEOUT | 请求超时(秒) | 60 |
| LLM_ENABLE_THINKING | 模型思考模式开关 | false |

#### 7.2.3 降级策略

当 `LLM_ENABLED=false` 或 API Key 未配置时，所有 Agent 自动走 fallback 逻辑：

- 消息面：返回原始新闻列表，情绪评分为 0（中性）
- 板块联动：返回原始板块数据，不做 AI 分析
- 宏观环境：返回原始指标数据
- 增强建议：回退到现有纯规则引擎（技术指标评分）

系统在无 AI 状态下仍可正常运行，前端展示"AI 分析不可用"提示。

### 7.3 Agent 定义

#### 7.3.1 消息面情绪分析 Agent

| 项目 | 说明 |
| ----- | ------ |
| 数据源 | AKShare `stock_news_em` 个股新闻 |
| 输入 | 股票代码、股票名称、最近 20 条新闻 |
| LLM 输出 | overall_sentiment(-1~1)、sentiment_label(利好/利空/中性)、key_news[]、analysis |
| 降级输出 | 原始新闻列表，sentiment=0 |

#### 7.3.2 板块联动分析 Agent

| 项目 | 说明 |
| ----- | ------ |
| 数据源 | AKShare 行业板块、板块成分股、概念板块 |
| 输入 | 股票代码、所属行业板块数据、板块内排名、概念板块 |
| LLM 输出 | sector_name、sector_trend(强势/弱势/震荡)、relative_strength(-1~1)、rotation_signal、analysis |
| 降级输出 | 原始板块数据和排名 |

#### 7.3.3 宏观环境感知 Agent

| 项目 | 说明 |
| ----- | ------ |
| 数据源 | 上证指数走势、北向资金、市场涨跌概况 |
| 输入 | 大盘近期走势、北向资金流向、市场涨跌家数 |
| LLM 输出 | market_phase(牛市/熊市/震荡市)、market_sentiment(-1~1)、risk_level(低/中/高)、analysis |
| 降级输出 | 原始宏观指标数据 |

#### 7.3.4 增强版买卖建议 Agent（链路终结点）

| 项目 | 说明 |
| ----- | ------ |
| 数据源 | 上游 3 个 Agent 结果 + 技术指标评分 + 用户画像 + 持仓信息 |
| 输入 | 四维度数据汇总 + 用户个人交易风格 |
| LLM 输出 | signal、confidence、reasoning[]、dimension_scores{technical,sentiment,sector,macro}、risk_warnings[]、position_advice、summary |
| 降级输出 | 现有 advice_service 纯规则引擎结果 |

### 7.4 Agent 协作数据流

```text
用户请求: POST /api/agent/enhanced-analysis
         body: {stock_code, stock_name}
                     │
        ┌────────────┼────────────┐
        │            │            │
        ▼            ▼            ▼
  Sentiment      Sector        Macro
   Agent          Agent         Agent
  (消息面)       (板块)        (宏观)
        │            │            │
        └────────────┼────────────┘
                     │ 3个结果汇总
                     ▼
          Enhanced Advice Agent
          + 技术指标 (advice_service)
          + 用户画像 (profile_service)
          + 持仓信息 (StockPosition)
                     │
                     ▼
              综合建议响应
```

- 前 3 个 Agent **并行执行**（ThreadPoolExecutor），减少总延时
- Enhanced Advice Agent **串行执行**，等待上游全部完成
- 每个 Agent 也可通过独立 API 端点单独调用

### 7.5 API 端点

| 端点 | 方法 | 描述 |
| ----- | ---- | ------ |
| /api/agent/sentiment | POST | 消息面情绪分析（body: stock_code, stock_name） |
| /api/agent/sector | POST | 板块联动分析 |
| /api/agent/macro | POST | 宏观环境分析 |
| /api/agent/enhanced-analysis | POST | 增强版综合建议（全链路） |
| /api/agent/llm-status | GET | LLM 配置状态查询 |
| /api/agent/reload-config | POST | 热重载 .env 配置 |
| /api/agent/cache/{stock_code} | DELETE | 清除 Agent 缓存 |

### 7.6 Agent 结果缓存

Agent 结果存储在 `AgentResultCache` 表中，避免重复 LLM 调用：

- 缓存 key：agent_name + stock_code + 日期
- 消息面 / 板块 / 宏观：当日有效
- 增强建议：1 小时有效
