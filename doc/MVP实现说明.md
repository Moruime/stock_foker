## 3 技术指标信号策略

买卖建议基于以下指标的综合评分（-1.5 ~ +1.5 分）：

| 指标 | 买入信号 | 卖出信号 |
| ------ | --------- | --------- |
| MACD | DIF 上穿 DEA（金叉 +1.5）; 多头排列 +0.5 | DIF 下穿 DEA（死叉 -1.5）; 空头排列 -0.5 |
| KDJ | K,D 均 < 20 超卖区 +1.0 | K,D 均 > 80 超买区 -1.0 |
| RSI | RSI < 30 超卖 +1.0 | RSI > 70 超买 -1.0 |
| 均线 | 价格 > MA5 > MA10 > MA20 多头排列 +1.0 | 价格 < MA5 < MA10 < MA20 空头排列 -1.0 |
| 布林带 | 价格触及下轨 +0.8 | 价格触及上轨 -0.8 |

综合评分 > 0.3 → 买入建议，< -0.3 → 卖出建议，其余 → 持有观望。

每条建议附带完整推理过程，包含各指标具体数值和判断逻辑。

## 4 数据源策略

采用 **AKShare 为主 + 新浪财经 API 为备** 的双数据源容灾 + 本地缓存架构：

```text
请求 → 本地 SQLite 缓存（命中？） → 直接返回
                ↓ (未命中/过期)
        AKShare/东方财富（主数据源）
                ↓ (失败)
        新浪财经 API（备用数据源）
                ↓
        写入 SQLite 缓存 → 返回
```

缓存策略：

- 缓存粒度：每只股票 + 每个周期（daily/weekly/monthly）
- 更新判断：最后缓存日期距今 > 1 天时触发增量更新
- 增量写入：只插入本地没有的新日期数据
- 当日覆盖：交易日内盘中数据允许覆盖更新
- 离线可用：远程失败但有缓存时直接返回缓存数据

## 6 已知限制

1. **东方财富 API 限流**：该数据源对自动化请求有频率限制，AKShare 部分接口可能偶发限流
2. **新浪接口数据量**：单次最多返回 300 条，长期历史需分批获取
3. **前端图表**：MACD/KDJ/RSI 指标数值已计算并传到前端，但在 K 线图中仅展示均线叠加，副图指标面板待后续优化
4. **炒股画像**：需要积累足够交易记录才有统计意义
5. **LLM 依赖**：AI 分析功能依赖外部 LLM API，未配置时自动降级为纯数据展示模式
