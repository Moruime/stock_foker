## 3 优化建议

### 3.1 性能优化

- Agent 响应时间：考虑流式返回（SSE），先展示已完成 Agent 结果，再展示综合建议
- LLM 调用优化：精简 Prompt 长度，减少 token 消耗和响应延迟
- data_fetcher 并行化：内部多个 AKShare 调用可并行化

### 3.2 工程质量

- 自动化测试：当前无测试覆盖，需补充 Agent 单元测试和 API 集成测试
- 错误监控：Agent 错误目前仅记录在日志和缓存表，缺少聚合监控
- TypeScript 严格模式：部分 Agent 返回数据使用 `Record<string, unknown>`，可细化类型

### 3.3 功能增强

- localStorage 持久化缓存：应用刷新后可恢复当日缓存，减少首次等待
  （当前已有 DB 缓存恢复，但 localStorage 未做）
- 多模型切换：Settings 页直接切换不同 LLM 模型，无需编辑 .env
- Prompt 版本管理：Agent Prompt 模板可外置为配置文件，便于迭代优化
- SnapshotPanel 与 Agent 刷新联动：刷新 Agent 后自动重载日期列表
