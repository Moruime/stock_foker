## 3 优化建议

### 3.1 性能优化

- SSE 流式返回：先展示已完成 Agent，再展示综合建议
- LLM Prompt 截断：数据过长时需精细截断策略
- Macro Agent 非缓存源也纳入缓存体系（4 个源当前走
  `parallel_fetch`，可统一为 `parallel_get_data_sources`）

### 3.2 工程质量

- 自动化测试：补充 Agent 单元测试和 API 集成测试
- 错误监控：Agent 错误仅在日志和缓存表，缺聚合监控
- TypeScript 严格化：部分 Agent 返回用
  `Record<string, unknown>`

### 3.3 数据源增强

- Prompt 版本管理：模板外置为配置文件
- 缓存清理策略：定期清理超过 7 天的历史缓存记录
- 熔断自动恢复：定时探测 API 可用性，自动重置熔断
