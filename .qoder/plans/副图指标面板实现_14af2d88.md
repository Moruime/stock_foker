# 副图指标面板实现

## 现状分析

- 后端 `stock_service.py` 的 `calculate_indicators` 已返回完整的 MACD(dif/dea/histogram)、KDJ(k/d/j)、RSI 数据
- 前端 `TechnicalIndicators` 类型已声明这些字段
- 当前 K 线图使用 2 层 ECharts grid：grid[0]=K线+均线(50%)，grid[1]=成交量(18%)
- 后端无需改动，所有指标数据已通过 API 返回

## 实现方案

采用「Segmented 切换」方式，在成交量下方添加第 3 层 grid，用户可切换 MACD / KDJ / RSI 三种视图。这样图表不会过高，交互清晰。

## Task 1: 添加副图指标切换控件和状态

**文件**: `frontend/src/pages/AnalysisPage.tsx`

- 新增 state: `const [subIndicator, setSubIndicator] = useState<'macd' | 'kdj' | 'rsi'>('macd');`
- 在 K 线 Card 内部、ReactECharts 上方，添加一个右对齐的 Segmented 切换器:
  - 选项: MACD / KDJ / RSI
  - 放在 Card title 的 extra 区域或图表上方右侧

## Task 2: 扩展 ECharts 图表配置

**文件**: `frontend/src/pages/AnalysisPage.tsx`

将当前 2 层 grid 扩展为 3 层，并调整高度分配:

```
grid[0]: K线主图  top: 8%,  height: 42%
grid[1]: 成交量    top: 56%, height: 14%
grid[2]: 副图指标  top: 74%, height: 14%
dataZoom slider:   top: 92%
```

对应新增:
- `grid[2]`: 副图区域
- `xAxis[2]`: 第 3 个分类轴（共享 dates）
- `yAxis[2]`: 第 3 个数值轴
- `dataZoom.xAxisIndex`: 扩展为 `[0, 1, 2]` 保证三层联动

## Task 3: 根据选中指标生成 series

**文件**: `frontend/src/pages/AnalysisPage.tsx`

根据 `subIndicator` 动态生成第 3 层 grid 的 series:

- **MACD**: 
  - DIF 线（蓝色）+ DEA 线（橙色）+ Histogram 柱状图（红涨绿跌）
  - legend: DIF, DEA, MACD柱

- **KDJ**:
  - K 线（蓝色）+ D 线（橙色）+ J 线（紫色）
  - 超买 80 / 超卖 20 参考线（markLine）

- **RSI**:
  - RSI 线（蓝色）
  - 超买 70 / 超卖 30 参考线（markLine）

颜色参考:
```
DIF / K / RSI: #4dabf7 (primary blue)
DEA / D:       #e8b339 (warning yellow)  
J:             #b388ff (purple)
```

**文件**: `frontend/src/pages/AnalysisPage.tsx`

- legend 中追加副图指标名称，且副图 legend 位置放在副图 grid 上方
