# 数据模块解耦与独立缓存

## 架构设计

当前问题：原始数据嵌入在 Agent 执行流程中，无法独立获取/缓存。

目标架构（三层分离）：

```
前端数据面板 ──→ /api/data-source/{stock_code}/{type} ──→ DataSourceCache ──→ hithink API
前端 Agent 分析 ──→ /api/agent/{type} ──→ AgentResultCache ──→ Agent(使用缓存的数据源)
```

数据源类型与归属页面：

| source_type | 数据内容 | 归属页面 |
|---|---|---|
| hithink_news | 同花顺财经资讯 | SentimentPage |
| announcements | 公司公告 | SentimentPage |
| industry_valuation | 行业 PE/PB/ROE | SectorPage |
| market_data | 资金流向/换手率 | SectorPage |
| hithink_index | 主要指数行情 | MacroPage |
| reports | 研报观点 | AnalysisPage |
| basicinfo | 公司基本资料 | AnalysisPage |
| business | 主营业务构成 | AnalysisPage |
| shareholders | 股东信息 | AnalysisPage |

前端组织方式：不新建大量组件文件，而是在每个页面内通过共享 `useDataSource` hook 独立管理各数据区块的加载/缓存状态，保持页面结构清晰不散乱。

## Task 3: 后端数据源缓存服务

**新建文件**: `backend/app/services/data_source_service.py`

核心逻辑：

```python
# 9 种数据源的获取函数映射
SOURCE_FETCHERS = {
    "hithink_news": lambda stock_name: fetch_hithink_news(stock_name),
    "announcements": lambda stock_name: fetch_hithink_announcements(stock_name),
    "industry_valuation": lambda stock_name: fetch_hithink_industry_data(stock_name),
    "market_data": lambda stock_name: fetch_hithink_market_data(stock_name),
    "hithink_index": lambda _: fetch_hithink_index_data(),
    "reports": lambda stock_name: fetch_hithink_reports(stock_name),
    "basicinfo": lambda stock_name: fetch_hithink_basicinfo(stock_name),
    "business": lambda stock_name: fetch_hithink_business_data(stock_name),
    "shareholders": lambda stock_name: fetch_hithink_shareholders(stock_name),
}

def get_data_source(db, stock_code, stock_name, source_type, force_refresh=False) -> dict:
    """获取数据源：优先返回缓存，未命中则调用 API 并写入缓存。"""
    if not force_refresh:
        cached = _get_cached_source(db, stock_code, source_type)
        if cached:
            return cached
    data = SOURCE_FETCHERS[source_type](stock_name)
    _save_source_cache(db, stock_code, source_type, data)
    return data
```

同时提供 `get_data_source_cached_only()` 用于仅查询缓存而不触发 API 调用。

## Task 6: 前端 useDataSource hook

**新建文件**: `frontend/src/hooks/useDataSource.ts`

```typescript
interface DataSourceResult {
  data: Record<string, unknown> | null;
  loading: boolean;
  error: string;
  timestamp: string;
  fromCache: boolean;
  refresh: () => void;
}

function useDataSource(stockCode: string | undefined, sourceType: string): DataSourceResult
```

功能：
- 组件挂载时自动获取数据（调用 GET /api/data-source/...）
- 内部使用 `AgentCacheContext` 或自建轻量内存缓存存储结果
- 返回 `{ data, loading, timestamp, fromCache, refresh }`
- `refresh()` 调用 POST 强制刷新端点

为复用内存缓存，扩展 `AgentCacheContext` 增加 `getSourceCache / setSourceCache` 方法（key 格式 `{stockCode}:source:{sourceType}`），或在 hook 内部用模块级 Map 实现（更简单）。

```typescript

export const refreshDataSource = (stockCode: string, sourceType: string, stockName: string) => ...
```
