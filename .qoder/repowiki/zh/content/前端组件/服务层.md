## 简介
本文件聚焦于 Stock Foker 应用的服务层，系统性梳理前端 API 封装与后端服务层的交互设计与实现细节。内容涵盖：
- HTTP 请求封装与错误处理机制
- 数据模型与类型定义体系
- API 接口调用方式、参数格式与返回值结构
- 与后端 API 的通信协议与数据同步机制
- 错误处理策略与重试机制
- **新增**：数据源服务层的统一数据访问接口
- **新增**：数据源缓存策略与熔断机制
- 使用示例与最佳实践

## 项目结构
服务层由四层组成：
- 前端服务层：以 axios 封装 HTTP 请求，统一暴露业务 API 方法，并与类型定义协同工作
- 后端路由层：FastAPI 路由定义，负责请求校验、参数解析与响应模型
- 后端服务层：具体业务逻辑（K线与指标计算、买卖建议生成、画像统计、**新增数据源管理**）
- 数据库：SQLite 引擎与 SQLAlchemy ORM，提供会话管理与表结构定义

## 架构总览
服务层采用"前端 API 封装 + 后端 FastAPI 路由 + 服务层业务逻辑"的分层设计。前端通过 axios 发起 HTTP 请求，后端路由层完成参数解析与响应模型校验，服务层执行具体算法与数据处理，数据库持久化存储。

```mermaid
sequenceDiagram
participant UI as "前端页面<br/>AnalysisPage.tsx/TradesPage.tsx"
participant API as "前端API<br/>api.ts"
participant Router as "后端路由<br/>stock_router.py/data_source_router.py"
participant Service as "服务层<br/>stock_service.py/advice_service.py/profile_service.py<br/>data_source_service.py"
participant DataFetcher as "数据获取层<br/>data_fetcher.py"
participant DB as "数据库<br/>database.py/models.py"
UI->>API : 调用 getStockAnalysis()/getDataSource()
API->>Router : GET /api/stocks/ : code/analysis 或 /api/data-source/{code}/{type}
Router->>Service : 调用业务逻辑(如 get_kline_data/generate_advice/generate_profile/get_data_source)
Service->>DataFetcher : 调用数据获取函数(fetch_hithink_*等)
DataFetcher->>外部API : 调用同花顺问财API
DataFetcher-->>Service : 返回原始数据
Service->>DB : 查询/写入(如 K线缓存/数据源缓存)
DB-->>Service : 返回数据
Service-->>Router : 返回结构化结果
Router-->>API : HTTP 响应(JSON)
API-->>UI : Promise.resolve(类型化数据)
```

## 性能考量
- 缓存策略：本地 SQLite 缓存 K 线和数据源，避免重复拉取；仅增量更新，减少网络与计算开销
- 指标计算：使用 pandas_ta 并对 Series 转换为列表，避免复杂对象传输
- 重试机制：对远程接口调用进行指数退避重试，提升稳定性
- **新增**：并行处理：数据源获取支持多线程并行，提高响应速度
- **新增**：熔断机制：外部 API 调用失败时自动熔断，避免雪崩效应
- 前端渲染：按需加载与懒渲染，避免一次性渲染大量数据

## 故障排查指南
- 前端错误处理
  - 在页面中捕获 Promise 拒绝，展示错误信息或空状态
  - 示例：分析页加载失败时显示错误提示
- 后端错误处理
  - 路由层对业务异常抛出 HTTPException，包含明确 detail
  - 服务层对外部接口失败抛出 RuntimeError，由路由层转换为 500
- **新增**：数据源错误处理
  - 熔断机制：当外部 API 返回 403 user limit 时自动熔断
  - 降级策略：API 失败时返回历史缓存数据
  - 并行错误：单个数据源失败不影响其他数据源获取
- 常见问题
  - 未关注股票：更新时间框架接口返回 404
  - 交易记录不存在：更新/删除接口返回 404
  - 外部数据源不可用：K线获取失败，优先返回缓存，否则抛出错误
  - **新增**：数据源类型无效：返回 400 错误

### 数据源类型与缓存策略
- **数据源类型注册表**

- **缓存策略**
  - 新鲜度边界：每天 09:00
  - 优先级：当日新鲜缓存 → API 实时调用 → 历史缓存回退

- **熔断机制**
  - 检测到用户限额（HTTP 403 user limit）时触发熔断
  - 熔断状态下跳过所有问财 API 调用
  - 支持手动重置熔断状态
